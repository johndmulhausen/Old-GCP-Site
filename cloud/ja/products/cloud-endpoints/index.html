Cloud Endpoints

RESTful サービスを作成して、iOS や Android、JavaScript クライアントにサービスを提供できます。クライアント ライブラリを自動的に生成して、フロントエンドと簡単に通信できます。組み込みの機能には、DoS 攻撃防御機能、OAuth 2.0 サポート、クライアント キー管理などがあります。

特長

1 つのツールで複数のクライアント

1 つのソースから Android、iOS、ウェブベースのクライアント用にクライアント ライブラリを作成します。Cloud Endpoints はコードをラップして API サーバー をほんの数ステップで作成します。Cloud Endpoint API ライブラリは、Java、Python、Go、PHP で利用できます。Cloud Endpoints に関する詳細はドキュメントをご覧になるか、サンプルの Tic Tac Toe ウェブ アプリケーションをお試しください。

App Engine インフラストラクチャの拡張

App Engine で利用可能なすべてのツールとライブラリが携帯端末で利用できるようになりました。App Engine バックエンドを使用するだけで、追加作業なしにデータストア、Cloud Storage、タスク キューにアクセスできます。OAuth 2.0 認証は、例を参考にすることで、すばやく統合できます。

メンテナンスの少ないクライアント/サーバー

Cloud Endpoints の後ろで App Engine が動くため、サーバーのメンテナンスや負荷分散を気にする必要がなく、スケーリングは時間も手間もかかりません。App Engine やその他の Google のクラウド サービス同様、支払いは従量課金制です。

柔軟なクライアント側の統合

サーバー側 API にアノテーションを付け、その後クライアント ライブラリを自動的に作成します。クライアント ライブラリは Android および iOS 用に作成されます。最小限の JavaScript クライアント ライブラリで、標準のウェブ クライアントを作動させます。すべてのクライアントが同様の API および同じバックエンドを使用するので、開発時間を抑えることができます。

料金

App Engine および Compute Engine で実行するアプリケーションについては、Cloud Endpoints は無料でご利用いただけます。

ドキュメントと関連情報

Cloud Endpoints を使用して App Engine アプリケーション用のクライアント API を自動的に作成します。迅速に立ち上げて稼働できるように、さまざまな App Engine ランタイム用にチュートリアルと手順が用意されています。

リソース

  • Endpoints API の作成

    Endpoints API はリモート プロシージャ コール（RPC）サービスで、外部クライアントからアクセス可能なリモート メソッドを提供します。各 Endpoints API は RPC サービス クラスで構成され、その RPC サービス クラスは、ProtoRPC remote.Service クラス、および 1 つ以上のメソッドをサブクラスとします。

  • クライアント ライブラリの生成

    エンドポイントの作成が完了したら、Android および iOS 用のクライアント ライブラリを生成できます。

  • iOS クライアントでの Endpoints の使用

    iOS アプリで Endpoints を使用するには、次のことを行う必要があります: クライアント ライブラリ生成ツールをコンパイルしてライブラリを生成します。必要なファイルを iOS プロジェクトに追加します。リクエストの実行に使用するサービス オブジェクトを作成します。Endpoint によって公開される API を呼び出します。オプションで、認証された呼び出しのサポートを追加します。
