Run Apache Hadoop easily on Google Cloud Platform

Apache Hadoop runs very well on Google Cloud Platform where it benefits from the quality of compute, storage and network infrastructure. In addition, Google has provided native connectors for Hadoop so that code running in Hadoop (via Java MapReduce, Hadoop Streaming, Pig, Hive, etc.) can directly access data stored in Google Cloud Storage, Google BigQuery and Google Cloud Datastore. This simplifies the architecture of Hadoop systems on Google, opens the door to more on-demand Hadoop clusters (which can take advantage of per-minute billing) and, as Accenture found in this study, improve performance.

Benefits of Hadoop on Google Cloud Platform:

  • Quick Startup times

    Compute Engine virtual machines start in seconds.

  • Fast I/O

    MapR crushed the MinuteSort record using Hadoop and Compute Engine.

  • Run at Scale

    Throw more CPUs at your computations without worrying about breaking the bank. Per-minute billing lets you optimize for scale and speed.

MapR Technologies

MapR used Google Compute Engine to set a new world record for MinuteSort, sorting 15 billion 100-byte records (a total of 1.5 trillion bytes) in 60 seconds. The benchmark, often referred to as the World Cup of data sorting, demonstrated how quickly data can be sorted starting and ending on disks. While the previous MinuteSort record was achieved with custom hardware, MapR set the record using Google Compute Engine, Hadoop MapReduce and the MapR Distribution for Apache Hadoop.
